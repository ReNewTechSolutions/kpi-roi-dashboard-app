## Bug Fix Policy
A “bug” is defined as:
- broken build/deploy due to repo code
- runtime error reproducible with documented setup steps
- RLS policies not functioning as intended (member/owner rules)

A “feature request” is:
- new modules or integrations
- UI redesign
- additional data models
- changes to business rules
